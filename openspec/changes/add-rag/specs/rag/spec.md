## ADDED Requirements

### Requirement: Local answer generation

Answers SHALL be generated by a local model through a `Generator` protocol; no
cloud API SHALL be used. If the local model is unavailable, generation SHALL fail
loudly rather than fall back to any remote service.

#### Scenario: Generation uses the injected local generator

- **WHEN** an ask request is served with a `Generator` injected
- **THEN** the answer is produced by that generator and no cloud call is made

### Requirement: Retrieval decides the sources

The candidate sources for an answer SHALL be chosen by the deterministic hybrid
retrieval (top-k), not by the model. The model SHALL receive only those sources
and SHALL be instructed to answer from them.

#### Scenario: Sources come from hybrid retrieval

- **WHEN** an ask request is served
- **THEN** the sources passed to the generator are exactly the top-k hybrid
  search hits for the query

### Requirement: Citations are verified against the sources

Every citation in the answer SHALL be verified to be one of the retrieved source
document ids; citations not in the retrieved set SHALL be dropped. An answer with
no valid citation SHALL be reported as ungrounded, never returned as fabricated
content.

#### Scenario: Invented citation is dropped

- **WHEN** the model returns a citation to a document id that was not among the
  sources
- **THEN** that citation is removed from the returned answer

#### Scenario: No valid citation yields an ungrounded result

- **WHEN** none of the model's citations match the retrieved sources
- **THEN** the result is reported as having no grounded answer, not as prose

### Requirement: Only de-identified text reaches the model

The sources handed to the model SHALL come from the de-identified store; clear
PII SHALL never be sent to the model. Hybrid retrieval returns hits carrying
`document_id`/score/`object_key`/vector but NOT passage text, so `Source.text`
SHALL be resolved separately — by reading `DocumentText.anonymized_text` per hit
via a session (reusing `get_anonymized_text`). Both that column and the index
`text` field hold only anonymized/pseudonymized text, so the PII invariant holds;
raw document text SHALL NOT be read.

#### Scenario: Model input carries no clear PII

- **WHEN** sources are built for a query
- **THEN** each source's text is the de-identified `DocumentText.anonymized_text`
  for its hit, not raw document text

### Requirement: Ask API

RAG SHALL be exposed as an opt-in `GET /ask?q=` endpoint returning the answer and
its verified citations.

#### Scenario: Ask endpoint returns answer with citations

- **WHEN** `GET /ask?q=<question>` is called
- **THEN** it returns the generated answer and the list of verified source ids
